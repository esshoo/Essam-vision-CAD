# V14 Render + DXF Export Fix

## الهدف
هذه النسخة مبنية على V10 + V13، وتصلح مشكلتين ظهرتا أثناء الاختبار:

1. عند الخروج من إدارة العناصر كان العنصر المحذوف يظهر مرة أخرى في العرض 2D، رغم أنه لا يظهر في 3D.
2. ملف DXF المصدر كان يفتح بشكل غير صحيح أو يظهر شاشة سوداء، لأن ملف DXF الناتج كان غير مناسب لبعض parsers أو كان بلا عناصر هندسية صالحة.

## ما تم تعديله

### EntityLayerEditor.js
- إذا كان هناك أي تعديل على مستوى Entity مثل حذف/إخفاء/نقل، يتم إبقاء `EntityRenderBridge` فعالًا حتى بعد إغلاق لوحة إدارة العناصر.
- هذا يمنع رجوع الرسم الأصلي المجمع من `x-viewer` فوق التعديلات.
- عند `Restore All` يتم إيقاف الـ managed render والرجوع للعرض الأصلي.

### ProjectExporter.js
- تم إعادة بناء DXF export كصيغة AutoCAD R12 ASCII DXF أكثر توافقًا.
- تمت إضافة أقسام TABLES / LTYPE / LAYER / STYLE / BLOCKS / ENTITIES.
- يتم رفض تصدير DXF فارغ بدل إنشاء ملف مكسور.
- يتم استخدام CRLF line endings.
- يتم تنظيف أسماء الطبقات والنصوص لتقليل مشاكل parsers.

### LayerRulesPanel.js
- عند إخفاء/إظهار طبقة من لوحة الطبقات يتم مزامنة `EntityRenderBridge` عبر `EntityLayerEditor` بدل rebuild خام فقط.

## اختبار سريع
1. افتح ملف DXF.
2. فعّل إدارة العناصر.
3. احذف عنصرًا واحدًا.
4. أغلق لوحة إدارة العناصر.
5. يجب ألا يرجع العنصر المحذوف بصريًا.
6. افتح 3D، يجب ألا يظهر العنصر.
7. صدّر DXF وافتحه مرة أخرى.
8. يجب أن يفتح بدون خطأ `No objects found`.
